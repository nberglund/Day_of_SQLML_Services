Part 1 - Streaming

Quite often when you either create models or score / predict against a model you can run into memory issues, as the data is loaded in memory. One advantage that sp_execute_external_script has, is that is supports streaming - data is streamd in batches. In this part of the lab we will work with this.

Open the streaming.sql source file in SSMS (the file is in the same directory as above). As you see there are three statements in the file, and they all look something like so:

DECLARE @model varbinary(max) = (SELECT TOP(1) ModelBin
                                 FROM dbo.tb_Model2
                                 WHERE ModelName = 'GLM_75Pct');  
  EXEC sp_execute_external_script @language = N'R',  
     @script = N'  
       mod <- unserialize(model); 

       cat(paste0("R Process ID = ", Sys.getpid()))
       cat("\n")
       cat(paste0("Nmbr rows = ", nrow(InputDataSet)))
       cat("\n")
       cat("----------------")
       cat("\n")

        OutputDataSet <- data.frame(predict(mod, 
                                  newdata = InputDataSet, 
                                  type = "response"))',  
  @input_data_1 = N' SELECT TOP(100) y, rand1, rand2, rand3, 
                      rand4, rand5 
              FROM dbo.tb_Rand_10M',  
  @params = N'@model varbinary(max)', 
  @model = @model  
WITH RESULT SETS ((Y_predict float));
GO

At the end of Part 1 we saw in Figure 4 that even though we streamed the data, we did so in the same process. We do know that SQL Server supports “out of the box” parallel execution, so let us see what we can do.

Open the parallel.sql source file in the before directory.

  1. The first statement does not have anything to do with SQL Server Machine Learning Services, but if you execute that and at the same time asks for the actual execution plan (Ctrl+M), you see something like so in the Execution plan tab:

NOTE: The reason for the OPTION(querytraceon 8649, MAXDOP 4) is twofold: querytraceon 8649 is me trying to force parallelism, and MAXDOP 4 is to set how many processors to use.
